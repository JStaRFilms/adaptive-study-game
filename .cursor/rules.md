## Key Rules
- **Server Components First**: Use `use client` only when necessary.
- **Zod Validation**: Validate ALL inputs.
- **Tailwind CSS**: Utility-first, no arbitrary values if possible.
- **Feature-Sliced Design**: `src/features/[feature]`, `src/components/ui`, `src/app`.
- **200-Line Rule**: Refactor if file exceeds 200 lines.
